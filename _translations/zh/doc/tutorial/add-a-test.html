现在您的代码已经达到了一个稳定的状态（顺便说一句，做得不错）， 是时候添加一个测试了。在开发过程中测试代码，可以发现在修改过程中引入的错误。 在本主题中，您将为 Hello 函数添加一个测试。

注意：本主题是多部分教程的一部分，该教程从 创建一个 Go 模块 开始。

Go 内置的单元测试支持使得边开发边测试变得更加容易。 具体来说，通过使用命名约定、Go 的 testing 包以及 go test 命令，您可以快速编写和执行测试。

  1. 在 greetings 目录下，创建一个名为 greetings_test.go 的文件。

    文件名以 _test.go 结尾会告诉 go test 命令 该文件包含测试函数。

  2. 在 greetings_test.go 中，粘贴以下代码并保存文件。 
    package greetings

import (
    "testing"
    "regexp"
)

// TestHelloName 调用 greetings.Hello 并传入一个名字，检查
// 返回值是否有效。
func TestHelloName(t *testing.T) {
    name := "Gladys"
    want := regexp.MustCompile(`\b`+name+`\b`)
    msg, err := Hello("Gladys")
    if !want.MatchString(msg) || err != nil {
        t.Errorf(`Hello("Gladys") = %q, %v, want match for %#q, nil`, msg, err, want)
    }
}

// TestHelloEmpty 用一个空字符串调用 greetings.Hello，
// 检查是否返回错误。
func TestHelloEmpty(t *testing.T) {
    msg, err := Hello("")
    if msg != "" || err == nil {
        t.Errorf(`Hello("") = %q, %v, want "", error`, msg, err)
    }
}

    在这段代码中，您：

    • 在与被测代码相同的 package 中实现测试函数。
    • 创建了两个测试函数来测试 greetings.Hello 函数。测试函数名遵循 TestName 形式， 其中 Name 描述了特定测试的内容。此外，测试函数 以一个指向 testing 包的 testing.T 类型 的 pointer 作为参数。您将使用此参数的方法来从您的测试中 进行报告和日志记录。
    • 实现了两个测试：
      • TestHelloName 调用 Hello 函数， 传入一个 name 值，函数应该能够返回一个有效的响应消息。 如果调用返回错误或意外的响应消息（即不包含您传入的名字）， 您可以使用 t 参数的 Errorf 方法 将消息打印到控制台。
      • TestHelloEmpty 用一个空字符串调用 Hello 函数。 此测试旨在确认您的错误处理是否正常工作。 如果调用返回了非空字符串或没有错误，您可以使用 t 参数的 Errorf 方法 将消息打印到控制台。
  3. 在 greetings 目录的命令行中，运行 go test 命令 来执行测试。

    go test 命令会执行测试文件（文件名以 _test.go 结尾） 中的测试函数（函数名以 Test 开头）。您可以添加 -v 标志来获取详细输出，该输出会列出所有测试及其结果。

    测试应该通过。 

    $ go test
PASS
ok      example.com/greetings   0.364s

$ go test -v
=== RUN   TestHelloName
--- PASS: TestHelloName (0.00s)
=== RUN   TestHelloEmpty
--- PASS: TestHelloEmpty (0.00s)
PASS
ok      example.com/greetings   0.372s
  4. 破坏 greetings.Hello 函数以查看失败的测试。

    TestHelloName 测试函数会检查您作为 Hello 函数参数指定的名称的返回值。为了查看失败的测试结果，请修改 greetings.Hello 函数，使其不再包含该名称。

    在 greetings/greetings.go 文件中，用以下代码替换 Hello 函数。请注意，高亮的行更改了函数返回的值， 就好像 name 参数被意外移除了一样。 

    // Hello 为指定的人返回一个问候。
func Hello(name string) (string, error) {
    // 如果没有提供名字，则返回带有消息的错误。
    if name == "" {
        return name, errors.New("empty name")
    }
    // 使用随机格式创建消息。
    // message := fmt.Sprintf(randomFormat(), name)
    message := fmt.Sprint(randomFormat())
    return message, nil
}
  5. 在 greetings 目录的命令行中，运行 go test 来执行测试。

    这次，不带 -v 标志运行 go test。 输出将只包含失败测试的结果，这在您有大量测试时会很有用。 TestHelloName 测试应该会失败——TestHelloEmpty 仍然会通过。

FAIL    example.com/greetings   0.182s
