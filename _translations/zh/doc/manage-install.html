  • 从源码安装 Go -- 如何检出源码、在您的机器上构建并运行它们。

安装多个 Go 版本

您可以在同一台机器上安装多个 Go 版本。例如，您可能想在多个 Go 版本上测试您的代码。有关可以通过此方式安装的版本列表，请参见下载页面。

注意： 要使用此处描述的方法进行安装，您需要先安装 git。

要安装额外的 Go 版本，请运行 go install 命令，并指定要安装版本的下载位置。以下示例使用版本 1.10.7 进行说明： 

$ go install golang.org/dl/go1.10.7@latest
$ go1.10.7 download

要使用新下载的版本运行 go 命令，请将版本号附加到 go 命令后面，如下所示： 

$ go1.10.7 version
go version go1.10.7 linux/amd64

当您安装了多个版本时，您可以查看每个版本的 GOROOT 值来发现它们的安装位置。例如，运行如下命令： 

$ go1.10.7 env GOROOT

要卸载已下载的版本，只需删除其 GOROOT 环境变量指定的目录以及对应的 goX.Y.Z 二进制文件即可。

卸载 Go

您可以使用本文描述的步骤从系统中移除 Go。

移除用户配置和数据

Go 将用户配置存储在用户配置目录下的 go 目录中， 该目录由 os.UserConfigDir 返回。 您也可以通过运行 go env GOENV 来找到包含此配置文件的目录。

Go 将中间构建产物存储在 go env GOCACHE 返回的目录中。 可以使用 go clean -cache 命令将其移除。

Go 将下载的依赖项存储在 go env GOMODCACHE 返回的目录中。 可以使用 go clean -modcache 命令将其移除。

Go 将通过 go install 安装的二进制文件存储在 Go bin 目录中， 该目录由 go env GOBIN 返回（默认为 $HOME/go/bin）。 可以通过删除该目录来移除这些文件。如果您曾将此目录添加到 PATH 环境变量中，也应将其移除。

  1. 移除 /usr/local/go。

    这是默认的安装目录。

  2. 从您的 PATH 环境变量中移除 /usr/local/go。

    在 Linux 和 FreeBSD 下，请编辑 /etc/profile 或 $HOME/.profile 文件。

  3. 如果您手动将 Go bin 目录（通常是 $HOME/go/bin）添加到了 shell 配置文件中（例如通过添加 export PATH="$PATH:$(go env GOPATH)/bin"），请移除该条目。

  4. 重启任何已打开的终端会话以使更改生效。

  1. 移除 /usr/local/go。

    这是默认的安装目录。

  2. 从您的 PATH 环境变量中移除 /usr/local/go。

    如果您使用 macOS 安装包安装了 Go，请通过移除 /etc/paths.d/go 文件来完成此操作。

  3. 如果您手动将 Go bin 目录（通常是 $HOME/go/bin）添加到了 shell 配置文件中（例如通过添加 export PATH="$PATH:$(go env GOPATH)/bin"），请移除该条目。

  4. 重启任何已打开的终端会话以使更改生效。

您可以通过以下任一方法从 Windows 中移除 Go：

方法 1：控制面板（最简单）

  1. 打开控制面板，双击 添加/删除程序。
  2. 选择 Go 编程语言，点击卸载，然后按照提示操作。

方法 2：命令行

通过运行以下命令使用命令行进行卸载：

msiexec /x go{{version}}.windows-{{cpu-arch}}.msi /q

注意：使用此过程会自动移除由原始安装创建的 Windows 环境变量。
