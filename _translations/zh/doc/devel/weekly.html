Process.Wait 方法已移除其 option 参数，相关的常量也已从包中删除。同时，Wait 函数也被移除；仅保留了 Process 类型的方法。

不可移植的 Waitmsg 类型已被可移植的 ProcessState 取代。

runtime 包导出的大量 API 已被移除，相关功能由其他包提供。使用 runtime.Type 接口或其具体实现类型的代码现在应使用 reflect 包。使用 runtime.Semacquire 或 runtime.Semrelease 的代码应使用通道（channel）或 sync 包中的抽象。

runtime.Alloc、runtime.Free 和 runtime.Lookup 函数（这是一套用于调试内存分配器的不安全 API）已无替代方案。

runtime.Cgocalls 和 runtime.Goroutines 函数已分别重命名为 runtime.NumCgoCall 和 runtime.NumGoroutine。

本次每周快照包含time、os和text/template包的更新。time和os包的变更具有重大意义且相互关联。使用time包、text/template包或os.FileInfo类型的代码需要相应调整。

在time包中，现在仅用time.Time单一类型表示时间。请注意time.Time应作为值类型使用，这与以往通常使用*time.Time（大型结构体指针）的旧代码不同。（请删除*号）所有此前接受*time.Time、1970年以来秒数的int64或1970年以来纳秒数的int64的函数，现在都应接受time.Time。特别当需要替代int64参数时，该类型能清晰体现数值含义。

无论您之前调用的是time.Seconds、time.Nanoseconds、time.LocalTime还是time.UTC，现在都应使用新的time.Now函数。

若此前代码形如：

       t0 := time.Nanoseconds()
       myFunction()
       t1 := time.Nanoseconds()
       delta := t1 - t0
       fmt.Printf("耗时 %.2f 秒\n", float64(t1-t0)/1e9)

现在可改写为：

       t0 := time.Now()
       myFunction()
       t1 := time.Now()
       delta := t1.Sub(t0)
       fmt.Printf("耗时 %s\n", delta)

此代码中delta变量属于新类型time.Duration，它替代了各种表示纳秒计数的int64参数（但非1970年以来的时间戳）。

gofix可完成上述转换及其他部分调整，但不会将显式int64类型重写为time.Time。运行gofix后，您很可能需要手动修改程序中的相关类型。建议始终通过版本控制系统的差异比对功能检查gofix所做的更改。

详情请参见：http://weekly.golang.org/pkg/time/

在os包中，FileInfo结构体被替换为FileInfo接口，允许操作系统之外的代码实现该接口。引用*os.FileInfo（旧结构体指针）的代码应改为引用os.FileInfo（新接口）。该接口仅包含以下方法：

       type FileInfo interface {
               Name() string       // 文件基础名称
               Size() int64        // 文件大小（字节）
               Mode() FileMode     // 文件模式位
               ModTime() time.Time // 修改时间
               IsDir() bool        // Mode().IsDir()的简写
       }

若需访问操作系统内核提供的底层stat_t结构，可将FileInfo对象断言为*os.FileStat类型，并通过其Sys字段获取*syscall.Stat_t：

       dev := fi.(*os.FileStat).Sys.(*syscall.Stat_t).Dev

当然，这种实现不具备跨操作系统可移植性。

gofix将自动把*os.FileInfo重写为os.FileInfo，并将fi.Name等表达式转换为fi.Name()方法调用。
