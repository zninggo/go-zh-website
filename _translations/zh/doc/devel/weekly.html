2012-03-13 (Go 1 正式版候选版 1)



此每周快照非常接近我们期望的 Go 1 发布内容。目前仍有一些小的文档问题需要解决，并且在发布前还应处理少量错误，但绝大多数 Go 程序应完全不受我们从现在到正式发布期间所做任何更改的影响。

如果您有兴趣帮助我们测试、渴望尝试 Go 1，或者只是好奇，那么此每周快照值得一试。我们很快将发布新的 App Engine Go 1 测试版 SDK，因此如果您是 App Engine 用户，也可以在那里进行尝试。

本周快照包括对 time、os 和 text/template 包的更改。其中，对 time 和 os 包的更改意义重大且相互关联。使用 time 包、text/template 包或 os 包的 FileInfo 类型的代码将需要修改。

在 time 包中，现在只有一个类型 time.Time 来表示时间。请注意，time.Time 应作为值使用，这与通常使用指向大型结构体的指针 `*time.Time` 的旧代码相反。（请去掉 `*`。）任何先前接受 `*time.Time`、自 1970 年以来的 int64 秒数或自 1970 年以来的 int64 纳秒数的函数，现在都应接受 time.Time。特别是作为 int64 的替代品，该类型很好地文档化了其值的含义。

无论您之前调用的是 time.Seconds、time.Nanoseconds、time.LocalTime 还是 time.UTC，替代它们的都是新函数 time.Now。

如果您之前编写了如下代码：
       t0 := time.Nanoseconds()
       myFunction()
       t1 := time.Nanoseconds()
       delta := t1 - t0
       fmt.Printf("That took %.2f seconds\n", float64(t1-t0)/1e9)
现在可以写成：
       t0 := time.Now()
       myFunction()
       t1 := time.Now()
       delta := t1.Sub(t0)
       fmt.Printf("That took %s\n", delta)
在此代码片段中，变量 delta 的类型是新类型 time.Duration，它取代了众多以纳秒计数（但不是自 1970 年起）的 int64 参数。

Gofix 可以完成上述转换以及其他一些转换，但它不会将显式的 int64 类型重写为 time.Time。在运行 gofix 后，您很可能需要编辑程序来更改这些类型。一如既往，请务必使用版本控制系统的 diff 功能查看 gofix 所做的更改。

详情请参见：http://weekly.golang.org/pkg/time/。

在 os 包中，FileInfo 结构体被替换为 FileInfo 接口，允许操作系统之外的代码实现该接口。引用 `*os.FileInfo`（指向旧结构体的指针）的代码现在应改为引用 os.FileInfo（新接口）。该接口仅包含几个方法：
       type FileInfo interface {
               Name() string       // 文件的基名
               Size() int64        // 长度（字节）
               Mode() FileMode     // 文件模式位
               ModTime() time.Time // 修改时间
               IsDir() bool        // Mode().IsDir() 的简写
       }
如果您需要访问操作系统内核提供的底层 stat_t，可以假设您持有的 FileInfo 实际上是一个 `*os.FileStat`，并且其 Sys 字段实际上是一个 `*syscall.Stat_t` 来访问它，例如：
       dev := fi.(*os.FileStat).Sys.(*syscall.Stat_t).Dev
当然，这在不同操作系统间不一定可移植。

Gofix 将为您处理将 `*os.FileInfo` 重写为 os.FileInfo 的工作，并且它也会将诸如 `fi.Name` 之类的表达式重写为诸如 `fi.Name()` 之类的调用。

详情请参见：http://weekly.golang.org/pkg/os/#FileInfo。模板包已更新，导出了新的、更简洁的 API。Set 类型已被移除。现在，模板通过一起解析自动关联；嵌套定义会隐式创建关联关系。只有相互关联的模板才能互相调用。这种方法大幅缩减了构造 API 的复杂度。html/template 包也相应更新。针对旧版 API 最简单常见的使用方式，gofix 提供了自动转换工具。未涉及 Set 类型的代码在运行 gofix 后很可能可直接使用；而使用了 Set 的代码则需手动修改。模板定义语言本身未作更改。

2011-10-26 (新 rune 类型)



此快照引入了 rune 类型，它是 int 的别名，应用于 Unicode 码点。

Go 的未来版本（Go 1 之后）将把 rune 改为 int32 而不是 int 的别名。持续使用 rune 是确保你的代码在此更改前后都能构建的方法。

要测试你的代码是否 rune 安全，你可以使用您可以执行以下命令，该命令会构建一个编译器，其中 rune 是 int32 的别名而非 int 的别名：

	GOEXPERIMENT=rune32 ./all.bash

此外，请在代码上运行 govet 来识别那些可能需要更新其签名的方法。
