本页概述了 Go 1 之前各稳定版本之间的变更内容。 近期发布的版本说明请参见版本发布历史页面。

r60 (发布于 2011/09/07)

r60 版本对应于 weekly.2011-08-17。 本节重点介绍该版本中最重要的变更。 更详细的摘要请参见 周发布说明。 完整信息请查阅 Mercurial 变更列表。

语言

现在要求 "else" 代码块必须使用花括号，除非 "else" 的主体是另一个 "if"。 由于 gofmt 总是会自动加上花括号，因此经 gofmt 格式化的程序不受影响。 要修复其他程序，请运行 gofmt。

包

http 包 的 URL 解析和查询转义代码 （例如 ParseURL 和 URLEscape）已移至新的 url 包，并对名称进行了一些简化。 客户端代码可以使用 gofix 自动更新。

image 包 中 image.RGBA 和 image.NRGBA 等结构体类型的 Pix 字段已发生重大变更。 不过，image.Image 接口类型并未改变， 如果您没有显式引用 Pix 字段，则无需修改代码。 例如，如果您使用 image/jpeg 包解码多个图像， 然后使用 image/draw 进行组合， 最后使用 image/png 编码结果，那么您的代码应该仍能正常工作。 如果您的代码确实引用了 Pix 字段，请参阅 weekly.2011-07-19 快照说明以了解如何更新代码。

template 包 已被一个新的模板包 （原 exp/template）所取代。原始的 template 包仍可作为 old/template 使用。 old/template 包已被弃用，后续将被移除。 Go 源码树已更新为使用新的 template 包。我们鼓励旧 template 包的用户切换到新包。 使用 template 或 exp/template 的代码需要分别将其导入行更改为 "old/template" 或 "template"。

工具

Goinstall 现在使用新的标签选择方案。 在下载或更新时，goinstall 会查找带有 "go." 前缀且与本地 Go 版本对应的标签或分支。 对于 Go release.r58，它会查找 go.r58。 对于 weekly.2011-06-03，它会查找 go.weekly.2011-06-03。 如果未找到特定的 go.X 标签或分支，它将选择最近的早期版本。 如果找到合适的标签或分支，goinstall 将使用该版本的代码。 否则，它将使用版本控制系统选择的默认版本。 我们鼓励库作者在其仓库中使用相应的标签或分支名称，以使其库更易于访问。

次要修订

r60.1 包含一个 链接器修复，一对 goplay 修复， 以及一个 json 包 修复 和一个新 结构体标签选项。

r60.2 修复 了一个涉及映射的内存泄漏。

r60.3 修复了一个 reflect 包的 bug。

r59 (发布于 2011/08/01)

r59 版本对应于 weekly.2011-07-07。 本节重点介绍该版本中最重要的变更。 更详细的摘要请参见 周发布说明。 完整信息请查阅 Mercurial 变更列表。

语言

本版本包含一项语言变更，限制了 goto 的使用。 本质上，位于代码块外部的 goto 语句不能跳转到该代码块内部的标签。 如果您的代码使用了 goto，可能需要修改。 请参见此变更集以了解新规则如何影响 Go 源码树。

包

和往常一样，gofix 将处理这些包 API 变更所需的大部分重写工作。

http 包 提供了一个新的 FileSystem 接口，用于访问文件。 FileServer 辅助函数现在接受一个 FileSystem 参数，而不是一个显式的文件系统根路径。 通过实现您自己的 FileSystem，您可以使用 FileServer 来提供任意数据。

os 包 的 ErrorString 类型已被隐藏。 os.ErrorString 的大多数用途可以用 os.NewError 替代。

reflect 包现在支持一种新的结构体标签方案，该方案允许多个包之间共享结构体标签。在此方案中，标签必须采用以下形式： 

	`key:"value" key2:"value2"`

StructField 类型的 Tag 字段现在的类型是 StructTag，它有一个 Get 方法。json 和 xml 的使用者需要更新代码。原本写作 

	type T struct {
		X int "name"
	}

的代码现在应该改为 

	type T struct {
		X int `json:"name"`  // 或 `xml:"name"`
	}

使用 govet 来识别那些需要更改为新语法的结构体标签。

sort 包的 IntArray 类型已重命名为 IntSlice，Float64Slice 和 StringSlice 也类似。

strings 包的 Split 函数本身已被拆分为 Split 和 SplitN。 SplitN 与原来的 Split 功能相同。 新的 Split 等同于将 SplitN 的最后一个参数设为 -1。

image/draw 包的 Draw 函数现在需要一个额外的参数，即合成操作符。 如有疑问，请使用 draw.Over。

工具

Goinstall 现在可以从任意远程仓库安装软件包和命令（而不仅仅是 Google Code、GitHub 等）。详情请参阅 goinstall 文档。

r58（发布于 2011/06/29）

r58 版本对应于 weekly.2011-06-09，并包含额外的错误修复。本节重点介绍此版本中最显著的更改。更详细的摘要，请参阅 周版发布说明。完整信息，请参阅 Mercurial 变更列表。

语言

此版本修复了 在错误使用 goto 的程序中使用未初始化内存 的问题。

软件包

一如既往，gofix 将处理这些包 API 变更所需的大部分重写工作。

http 包的 Client.Get 方法移除了 finalURL 返回值。该值现在可通过 http.Response 上的新 Request 字段获得。大多数类型为 map[string][]string 的实例已被新的 Values 类型取代。

exec 包已重新设计，提供更方便和简洁的 API。

strconv 包的 Quote 函数现在只转义那些未被 unicode.IsPrint 分类为可打印的 Unicode 码点。之前，Quote 会转义所有非 ASCII 字符。这也影响了 fmt 包的 "%q" 格式指令。之前的引号行为仍然可以通过 strconv 的新函数 QuoteToASCII 获得。

os/signal 包的 Signal 和 UnixSignal 类型已移至 os 包。

image/draw 包是 exp/draw 的新名称。来自 exp/draw 的 GUI 相关代码现在位于 exp/gui 包。

工具

Goinstall 现在会观察 GOPATH 环境变量，以便在 Go 源码树之外构建和安装你自己的代码及外部库（并避免编写 Makefile）。

次要修订

r58.1 增加了 构建 和 运行时 的更改，以使 Go 能在 OS X 10.7 Lion 上运行。

r57（发布于 2011/05/03）

r57 版本对应于 weekly.2011-04-27，并包含额外的错误修复。本节重点介绍此版本中最显著的更改。更详细的摘要，请参阅 周版发布说明。完整信息，请参阅 Mercurial 变更列表。

新的 gofix 工具可以找到使用旧 API 的 Go 程序，并将其重写为使用较新的 API。在你更新到新的 Go 版本后，gofix 可以帮助你对程序进行必要的更改。Gofix 将处理下面描述的 http、os 和 syscall 包的更改，我们也会更新该程序以跟上未来库的更改。Gofix 并非总能完美处理所有情况，因此在提交更改之前，请阅读并测试它所做的更改。更多信息请参阅 gofix 博文。

语言

多重赋值语法取代了 closed 函数。用于通道接收的语法现在允许一个可选的第二个赋值变量，它是一个布尔值，指示该通道是否已关闭。以下代码： 

	v := <-ch
	if closed(ch) {
		// channel is closed
	}

现在应该写为：

	v, ok := <-ch
	if !ok {
		// channel is closed
	}

未使用的标签现在被视为非法，这与未使用的局部变量一样。

包

gob 包现在可以编码和解码实现了 GobEncoder 和 GobDecoder 接口的类型的值。这允许包含未导出字段的类型传输自洽的描述信息；示例包括 big.Int 和 big.Rat。

http 包已经过重新设计。 对于客户端，引入了新的 Client 和 Transport 抽象层，它们能更好地控制 HTTP 细节，例如发送的请求头和遵循的重定向策略。这些抽象层使得实现自定义客户端（例如添加 OAuth2 功能）变得容易。 对于服务器端，ResponseWriter 移除了非核心方法。 Hijack 和 Flush 方法不再是必需的； 代码可以通过检查特定值是否实现了 Hijacker 或 Flusher 接口来测试它们。 RemoteAddr 和 UsingTLS 方法被替换为 Request 的 RemoteAddr 和 TLS 字段。 SetHeader 方法被 Header 方法替代； 其结果（类型为 Header） 实现了 Set 及其他方法。

net 包 移除了 Dial 的 laddr 参数， 并移除了 LookupHost 的 cname 返回值。 其实现现在尽可能使用 cgo 来通过 C 库的 getaddrinfo(3) 函数实现网络名称解析。这确保了 Go 和 C 程序以相同方式解析名称，同时也避免了 OS X 应用层防火墙的问题。

os 包 引入了简化的 Open 和 Create 函数。 原来的 Open 现在作为 OpenFile 使用。 StartProcess 的最后三个参数 已被替换为指向 ProcAttr 的指针。

reflect 包已经过重新设计。 Type 现在是一个实现了 所有可能类型方法的接口。 不再对 Type t 进行类型开关，而是对 t.Kind() 进行开关。 Value 现在是一个结构体值，它 实现了所有可能的值方法。 不再对 Value v 进行类型开关，而是对 v.Kind() 进行开关。 Typeof 和 ValueOf 现在被称为 TypeOf 和 ValueOf。 要创建一个可写的 Value，使用 New(t).Elem() 而不是 Zero(t)。 详情请参见 变更说明。 新的 API 允许更高效的 Value 实现， 避免了先前 API 所需的许多内存分配。

请记住，gofix 将处理这些包 API 更改所需的大部分代码重写工作。

工具

Gofix，一个新命令，上文已有描述。

Gotest 现在是一个 Go 程序，而不是一个 shell 脚本。 新的 -test.short 标志结合 testing 包的 Short 函数 允许你编写可以在普通或“短”模式下运行的测试； all.bash 在短模式下运行测试以减少安装时间。 Makefiles 了解该标志：使用 make testshort。

运行时支持现在实现了 CPU 和内存分析。 Gotest 的新 -test.cpuprofile 和 -test.memprofile 标志使得对测试进行分析变得容易。 要为你的 Web 服务器添加分析功能，请参见 http/pprof 文档。 其他用途，请参见 runtime/pprof 文档。

小版本修订

r57.1 修复了 http.FormFile 中的空指针解引用问题。

r57.2 修复了 在误用 goto 的程序中使用未初始化内存的问题。

r56 版本是第一个稳定版本，对应于 weekly.2011-03-07.1。 编号从 56 开始，因为在此版本之前， 我们如今所认为的每周快照当时被称为正式发布。
