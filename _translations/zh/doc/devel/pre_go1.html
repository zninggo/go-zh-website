本页总结了Go 1之前稳定版本之间的变更情况。 近期版本的发布说明请参见发布历史页面。

r60（发布于2011年9月7日）

r60版本对应于 weekly.2011-08-17。 本节重点介绍此版本中最重大的变更。 更详细的摘要，请参阅 周发布说明。 完整信息，请查阅 Mercurial 变更列表。

语言

现在，“else”代码块必须使用花括号，除非“else”的主体是另一个“if”。由于 gofmt 总是会自动添加这些花括号， 经过 gofmt 格式化的程序将不受影响。 要修复其他程序，请运行 gofmt。

包

http 包中的 URL 解析和查询转义代码 （如 ParseURL 和 URLEscape）已被移至 新的 url 包，并对名称进行了一些简化。 客户端代码可以通过 gofix 自动更新。

image 包中结构体类型（如 image.RGBA 和 image.NRGBA）的 Pix 字段已发生重大变更。 但 image.Image 接口类型未变， 因此如果你的代码没有显式引用 Pix 字段，则无需修改。例如，如果你使用 image/jpeg 包解码多张图像，使用 image/draw 进行合成，然后使用 image/png 编码结果，那么你的代码应该仍然可以像以前一样运行。 如果你的代码确实引用了 Pix 字段，请参阅 weekly.2011-07-19 快照说明以了解如何更新代码。

template 包已被一个新的 模板包（原 exp/template）取代。原来的 template 包仍可作为 old/template 使用。 old/template 包已被弃用，并将被移除。 Go 源代码树已更新为使用新的模板包。我们鼓励 旧模板包的用户切换到新包。使用 template 或 exp/template 的代码需要将其 导入行分别更改为 "old/template" 或 "template"。

工具

Goinstall 现在使用新的标签选择方案。 在下载或更新时，goinstall 会查找带有 "go." 前缀且对应于本地 Go 版本的标签或分支。对于 Go release.r58，它会查找 go.r58。对于 weekly.2011-06-03，它会查找 go.weekly.2011-06-03。 如果未找到特定的 go.X 标签或分支，它会选择 最接近的较早版本。如果找到合适的标签或分支，goinstall 会使用该版本的代码。否则，它将使用版本控制系统选择的默认版本。鼓励库作者在他们的代码库中使用 相应的标签或分支名称，以便于用户使用他们的库。

次要修订

r60.1 包括一个 链接器 修复，一对 goplay 修复， 以及一个 json 包 修复 和一个 新的 结构体标签 选项。

r60.2 修复 了一个涉及 map 的内存泄漏问题。

r60.3 修复了一个 reflect 错误。

r59（发布于2011年8月1日）

r59版本对应于 weekly.2011-07-07。 本节重点介绍此版本中最重大的变更。 更详细的摘要，请参阅 周发布说明。 完整信息，请查阅 Mercurial 变更列表。

语言

此版本包含一项限制 goto 使用的语言变更。本质上， goto 语句在代码块外部时，不能跳转到该代码块内部的标签。如果你的代码使用了 goto，可能需要进行修改。 请参见这个 变更集，了解新规则如何影响 Go 源代码树。

包

和往常一样，gofix 将处理这些包 API 变更所需的大部分重写工作。

http 包新增了一个 FileSystem 接口，用于提供文件访问。 FileServer 辅助函数现在接受一个 FileSystem 参数，而不是一个显式的文件系统根路径。通过 实现你自己的 FileSystem，你可以使用 FileServer 来提供任意数据的服务。

os 包的 ErrorString 类型已被隐藏。 大多数 os.ErrorString 的用法可以替换为 os.NewError。

reflect 包支持一种新的结构体标签方案，允许在多个包之间共享结构体标签。该方案中，标签必须采用以下形式： 

	`key:"value" key2:"value2"`

StructField 类型的 Tag 字段现在具有 StructTag 类型，该类型提供 Get 方法。json 和 xml 包的使用者需要更新代码。以下代码： 

	type T struct {
		X int "name"
	}

应改为： 

	type T struct {
		X int `json:"name"`  // 或 `xml:"name"`
	}

使用 govet 工具可识别需要更改为新语法的结构体标签。

sort 包的 IntArray 类型已重命名为 IntSlice，同样重命名的还有 Float64Slice 和 StringSlice。

strings 包的 Split 函数本身已被拆分为 Split 和 SplitN。SplitN 与旧版的 Split 功能相同。新版 Split 等同于将 SplitN 的最后一个参数设为 -1。

image/draw 包的 Draw 函数现在增加了一个额外参数——合成操作符。如有疑问，请使用 draw.Over。

工具

Goinstall 现在可从任意远程仓库安装包和命令（不再局限于 Google Code、GitHub 等）。详情请参阅 goinstall 文档。

r58（发布于 2011/06/29）

r58 版本对应于 weekly.2011-06-09，并包含额外的错误修复。本节重点说明该版本中最显著的变更。更详细的摘要请参阅 周度发布说明。完整信息请参阅 Mercurial 变更列表。

语言

此版本修复了 滥用 goto 的程序中未初始化内存的使用问题。

包

与往常一样，gofix 将处理包 API 变更所需的大部分代码重写。

http 包从 Client.Get 方法中移除了 finalURL 返回值。该值现在可通过 http.Response 上的新 Request 字段获取。大多数 map[string][]string 类型的实例已被替换为新的 Values 类型。

exec 包已重新设计，提供更便捷、更简洁的 API。

strconv 包的 Quote 函数现在仅转义那些未被 unicode.IsPrint 归类为可打印的 Unicode 码点。此前 Quote 会转义所有非 ASCII 字符。这也影响了 fmt 包的 "%q" 格式化指令。之前的引号行为仍可通过 strconv 的新函数 QuoteToASCII 实现。

os/signal 包的 Signal 和 UnixSignal 类型已移至 os 包。

image/draw 包是 exp/draw 的新名称。exp/draw 中的 GUI 相关代码现位于 exp/gui 包中。

工具

Goinstall 现在会遵循 GOPATH 环境变量，以在 Go 源代码树之外构建和安装你自己的代码及外部库（同时避免编写 Makefile）。

小版本修订

r58.1 增加了 构建 和 运行时 变更，使 Go 能够运行在 OS X 10.7 Lion 上。

r57（发布于 2011/05/03）

r57 版本对应于 weekly.2011-04-27，并包含额外的错误修复。本节重点说明该版本中最显著的变更。更详细的摘要请参阅 周度发布说明。完整信息请参阅 Mercurial 变更列表。

新的 gofix 工具可以查找使用旧 API 的 Go 程序并将其重写以使用更新的 API。在更新到新的 Go 版本后，gofix 可帮助你的程序进行必要的更改。Gofix 将处理下文描述的 http、os 和 syscall 包的变更，我们会持续更新该程序以跟进未来库的变更。 Gofix 无法完美处理所有情况，因此在提交前请阅读并测试它所做的更改。 更多信息请参阅 gofix 博客文章。

语言

多重赋值语法取代了 closed 函数。channel（通道）接收的语法允许一个可选的第二赋值，这是一个布尔值，用于指示 channel 是否已关闭。以下代码： 

	v := <-ch
	if closed(ch) {
		// channel 已关闭
	}

现在应写成：

	v, ok := <-ch
	if !ok {
		// channel 已关闭
	}

未使用的标签现在被视为非法，与未使用的局部变量相同。

包

gob 包现在能够编码和解码实现了 GobEncoder 和 GobDecoder 接口的类型的值。这允许具有未导出字段的类型传输自描述信息；例如 big.Int 和 big.Rat。

http 包已重新设计。 对于客户端，新增了 Client 和 Transport 抽象层，提供了对 HTTP 细节（如发送的标头和遵循的重定向）更多的控制。这些抽象使得实现添加 OAuth2 等功能的自定义客户端变得容易。 对于服务器端，ResponseWriter 移除了其非必要方法。 Hijack 和 Flush 方法不再是必需的； 代码可以通过检查特定值是否实现了 Hijacker 或 Flusher 来测试它们是否存在。 RemoteAddr 和 UsingTLS 方法已被 Request 的 RemoteAddr 和 TLS 字段取代。 SetHeader 方法已被 Header 方法取代； 其结果（类型为 Header）实现了 Set 和其他方法。

net 包从 Dial 中移除了 laddr 参数，并从 LookupHost 中移除了 cname 返回值。 其实现现在在可能的情况下使用 cgo，通过 C 库的 getaddrinfo(3) 函数来执行网络名称查找。这确保了 Go 和 C 程序以相同的方式解析名称，并避免了 OS X 应用程序级防火墙的问题。

os 包引入了简化的 Open 和 Create 函数。 原来的 Open 现在以 OpenFile 的形式提供。 StartProcess 的最后三个参数已被替换为一个指向 ProcAttr 的指针。

reflect 包已重新设计。 Type 现在是一个实现了所有可能类型方法的接口。 现在是对 t.Kind() 进行类型开关判断，而不是对 Type t 进行类型开关判断。 Value 现在是一个结构体值，实现了所有可能的值方法。 现在是对 v.Kind() 进行类型开关判断，而不是对 Value v 进行类型开关判断。 Typeof 和 NewValue 现在称为 TypeOf 和 ValueOf。 要创建可写的 Value，请使用 New(t).Elem() 而不是 Zero(t)。 有关完整细节，请参阅 变更描述。 新的 API 允许更高效地实现 Value，避免了旧 API 所需的许多内存分配。

请记住，gofix 将处理这些包 API 变更所需的大部分代码重写工作。

工具

Gofix，一个新命令，已在上文描述。

Gotest 现在是一个 Go 程序，而不是一个 shell 脚本。 新的 -test.short 标志与 testing 包的 Short 函数结合使用，允许您编写可以在正常模式或“短”模式下运行的测试； all.bash 以短模式运行测试以减少安装时间。 Makefiles 了解此标志：使用 make testshort。

运行时支持现在实现了 CPU 和内存分析。 Gotest 新增的 -test.cpuprofile 和 -test.memprofile 标志 使得对测试进行分析变得简单。 要将分析添加到您的 Web 服务器，请参阅 http/pprof 文档。 对于其他用途，请参阅 runtime/pprof 文档。

次要修订

r57.1 修复了 http.FormFile 中的空指针解引用。

r57.2 修复了 在错误使用 goto 的程序中出现的未初始化内存使用。

r56 (发布于 2011/03/16)

r56 版本是第一个稳定版本，对应于 weekly.2011-03-07.1。 版本号从 56 开始，因为在此版本之前，我们现在所称的每周快照曾被称为发布版本。
