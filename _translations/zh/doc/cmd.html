Go 语言提供了一套用于构建和处理 Go 源代码的程序集合。这些程序通常不直接运行，而是通过 go 程序来调用。

运行这些程序最常见的方式是作为 go 程序的子命令，例如 go fmt。以这种方式运行时，命令会处理完整的 Go 源代码包，go 程序会以适合包级别处理的参数调用底层二进制程序。

这些程序也可以作为独立的二进制文件运行，使用未修改的参数，通过 go 的 tool 子命令，例如 go tool cgo。对于大多数命令，这主要用于调试。部分命令，如 pprof，只能通过 go 的 tool 子命令访问。

Go 的安装过程还会安装一个名为 gofmt 的可执行文件，它等同于 go fmt，因为后者被频繁引用。

名称      概要
go      go 程序用于管理 Go 源代码并运行此处列出的其他命令。详细用法请参见命令文档。
cgo      Cgo 使得创建能够调用 C 代码的 Go 包成为可能。
cover      Cover 是一个用于创建和分析由 "go test -coverprofile" 生成的覆盖率分析结果的程序。
fix      Fix 用于查找使用旧版语言和库特性的 Go 程序，并将它们重写为使用新特性。
fmt      Fmt 用于格式化 Go 包。它也可以作为独立的 gofmt 命令使用，并提供更多通用选项。
doc      Doc 用于提取和生成 Go 包的文档。
vet      Vet 用于检查 Go 源代码并报告可疑的构造，例如参数与格式化字符串不匹配的 Printf 调用。

此处为节选列表。请参阅 完整命令参考 以获取编译器等更多内容的文档。
