有一系列程序用于构建和处理Go源代码。这些程序通常不是直接运行，而是通过go程序来调用。

运行这些程序最常见的方式是将其作为go程序的子命令，例如go fmt。以这种方式运行时，命令会处理完整的Go源代码包，go程序会使用适合包级别处理的参数来调用底层二进制程序。

这些程序也可以作为独立的二进制文件运行，使用未经修改的参数，通过go的tool子命令调用，例如go tool cgo。对于大多数命令来说，这主要用于调试。有些命令，如pprof，只能通过go的tool子命令访问。

Go安装过程还会安装一个名为gofmt的可执行文件，它等同于go fmt，因为它经常被引用。

点击链接可以查看更多文档、调用方式和使用详情。

名称      概述
go      go程序管理Go源代码并运行此处列出的其他命令。有关使用详情，请参阅命令文档。
cgo      Cgo支持创建调用C代码的Go包。
cover      Cover是一个用于创建和分析"go test -coverprofile"生成的覆盖率配置文件的程序。
fix      Fix查找使用旧版语言和库特性的Go程序，并将其重写为使用新特性。
fmt      Fmt格式化Go包，它也可以作为独立的gofmt命令使用，具有更通用的选项。
doc      Doc提取并生成Go包的文档。
vet      Vet检查Go源代码并报告可疑的构造，例如参数与格式字符串不匹配的Printf调用。

这是一个精简列表。有关编译器及更多内容的文档，请参阅完整命令参考。
