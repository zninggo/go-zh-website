简介

本教程涵盖以下内容：

  • 创建一个包含加载和保存方法的数据结构
  • 使用 net/http 包构建Web应用程序
  • 使用 html/template 包处理HTML模板
  • 使用 regexp 包验证用户输入
  • 使用闭包

所需预备知识：

  • 编程经验
  • 理解基本的Web技术（HTTP、HTML）
  • 一些UNIX/DOS命令行知识

main 函数首先调用了 http.HandleFunc，该函数告知 http 包使用 handler 来处理所有指向 Web 根路径 ("/") 的请求。

随后它调用了 http.ListenAndServe，指定它应在任意网络接口 (":8080") 的 8080 端口上进行监听。（暂时无需担心它的第二个参数 nil。） 此函数将阻塞运行，直至程序终止。

http://localhost:8080/monkeys

程序将显示一个包含以下内容的页面：

引入函数字面量与闭包

在每个处理函数中捕获错误条件引入了大量重复的代码。 如果我们能把每个处理函数都包装在一个进行此验证和错误检查的函数中呢？Go 语言的函数字面量提供了一种强大的功能抽象手段，可以在这里帮助我们。

首先，我们将每个处理函数的定义改写为接受一个 title 字符串： 

func viewHandler(w http.ResponseWriter, r *http.Request, title string)
func editHandler(w http.ResponseWriter, r *http.Request, title string)
func saveHandler(w http.ResponseWriter, r *http.Request, title string)

现在，我们定义一个包装函数，它接受一个上述类型的函数，并返回一个类型为 http.HandlerFunc 的函数（适合传递给 http.HandleFunc 函数）： 

func makeHandler(fn func (http.ResponseWriter, *http.Request, string)) http.HandlerFunc {
    return func(w http.ResponseWriter, r *http.Request) {
        // 在这里，我们将从 Request 中提取页面标题，
        // 并调用提供的处理函数 'fn'
    }
}

返回的函数被称为闭包，因为它封闭了在其外部定义的值。在本例中，变量 fn（makeHandler 的唯一参数）被闭包封闭了。变量 fn 将是我们的 save、edit 或 view 处理函数之一。

现在，我们可以取 getTitle 中的代码并在这里使用它（稍作修改）：

{{code "final.go" `/func makeHandler/` `/^}/`}}

makeHandler 返回的闭包是一个接受 http.ResponseWriter 和 http.Request 的函数（换句话说，就是一个 http.HandlerFunc）。 该闭包从请求路径中提取 title，并使用 validPath 正则表达式对其进行验证。如果 title 无效，则使用 http.NotFound 函数将错误写入 ResponseWriter。 如果 title 有效，则使用 ResponseWriter、Request 和 title 作为参数，调用被封闭的处理函数 fn。
