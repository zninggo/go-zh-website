save 方法返回一个 error 值，因为这是 WriteFile（一个将字节切片写入文件的标准库函数）的返回类型。save 方法返回错误值，以便在写入文件时出现任何问题时，应用程序可以处理它。如果一切顺利，Page.save() 将返回 nil（指针、接口和某些其他类型的零值）。

八进制整数字面量 0600 作为第三个参数传递给 WriteFile，表示应创建具有仅当前用户读写权限的文件。（详情请参阅Unix手册页 open(2)。）

{{code "http-sample.go"}}

然后它调用 http.ListenAndServe，指定它应在任意网络接口的 8080 端口上监听 (":8080")。（暂时不用担心它的第二个参数 nil。） 此函数将阻塞，直到程序终止。

ListenAndServe 总是返回一个错误，因为它只在发生意外错误时返回。 为了记录该错误，我们用 log.Fatal 包装了该函数调用。

一个 http.Request 是一个代表客户端 HTTP 请求的数据结构。r.URL.Path 是请求 URL 的路径部分。尾部的 [1:] 意思是"创建一个从第 1 个字符到末尾的 Path 子切片。"这将从路径名称中去掉开头的 "/"。

如果你运行此程序并访问以下 URL： 

http://localhost:8080/monkeys

该程序将显示一个包含以下内容的页面：

html/template 包是 Go 标准库的一部分。 我们可以使用 html/template 将 HTML 保存在单独的文件中，从而允许我们在不修改底层 Go 代码的情况下更改编辑页面的布局。

介绍函数字面量和闭包

在每个处理程序中捕获错误条件引入了大量重复代码。 如果我们能够将每个处理程序包装在一个执行此验证和错误检查的函数中呢？Go 语言的 函数字面量 提供了一种强大的抽象功能手段，可以在这里帮助我们。

首先，我们重写每个处理程序的函数定义，使其接受一个标题字符串： 

func viewHandler(w http.ResponseWriter, r *http.Request, title string)
func editHandler(w http.ResponseWriter, r *http.Request, title string)
func saveHandler(w http.ResponseWriter, r *http.Request, title string)

现在让我们定义一个包装函数，它接受一个具有上述类型的函数，并返回一个类型为 http.HandlerFunc 的函数（适合传递给 http.HandleFunc 函数）： 

func makeHandler(fn func (http.ResponseWriter, *http.Request, string)) http.HandlerFunc {
    return func(w http.ResponseWriter, r *http.Request) {
        // 在这里，我们将从请求中提取页面标题，
        // 并调用提供的处理程序 'fn'
    }
}

返回的函数被称为闭包，因为它封闭了在其外部定义的值。在这个例子中，变量 fn（makeHandler 的唯一参数）被闭包封闭了。变量 fn 将是我们的保存、编辑或查看处理程序之一。

现在我们可以从 getTitle 中提取代码并在此处使用（并做一些小的修改）：

{{code "final.go" `/func makeHandler/` `/^}/`}}

makeHandler 返回的闭包是一个接受 http.ResponseWriter 和 http.Request 的函数（换句话说，就是一个 http.HandlerFunc）。 该闭包从请求路径中提取 title，并使用 validPath 正则表达式对其进行验证。如果 title 无效，将使用 http.NotFound 函数向 ResponseWriter 写入一个错误。如果 title 有效，则会以 ResponseWriter、Request 和 title 作为参数调用封闭的处理程序函数 fn。
