当您希望让其他开发者能够使用某个模块时，需要将其发布以便Go工具可以识别它。一旦模块发布成功，导入该模块包的开发者就能通过执行`go get`等命令来解析对该模块的依赖。

> **注意：** 模块发布后，请勿修改已打标签的版本。对于使用该模块的开发者，Go工具会通过首次下载的副本对后续下载的模块进行验证。如果两者存在差异，Go工具将返回安全错误。与其修改已发布版本的代码，不如直接发布新版本。

**参见**

* 关于模块开发的概述，请参见[开发和发布模块](developing)
* 关于高级模块开发流程（包含发布环节），请参见[模块发布与版本管理工作流](release-workflow)。

## 发布步骤

请按照以下步骤发布模块：

1. 打开命令提示符，并切换到本地仓库中模块的根目录。

1. 运行`go mod tidy`命令，该命令会移除模块中可能积累的、不再需要的依赖项。```
    $ go mod tidy
    ```1.  最后一次运行 `go test ./...` 命令，确保一切工作正常。

    这会运行您使用 Go 测试框架编写的单元测试。```
    $ go test ./...
    ok      example.com/mymodule       0.015s
    ```1.  使用 `git tag` 命令为项目打上新版本号的标签。

    对于版本号，请使用一个能够向用户表明本次发布所包含变更性质的数字。更多信息，请参阅[模块版本编号](version-numbers)。```
    $ git commit -m "mymodule: changes for v0.1.0"
    $ git tag v0.1.0
    ```1.  将新标签推送到远程仓库。```
    $ git push origin v0.1.0
    ```1. 通过执行 [`go list` 命令](/cmd/go/#hdr-List_packages_or_modules) 使模块可用，该命令会提示 Go 更新其模块索引，以包含您正在发布的模块信息。

    在命令前需声明设置 `GOPROXY` 环境变量为某个 Go 代理。这将确保您的请求能正确到达代理服务器。```
    $ GOPROXY=proxy.golang.org go list -m example.com/mymodule@v0.1.0
    ```对您的模块感兴趣的开发者可以像使用其他任何模块一样，导入其中的包并运行 [`go get` 命令]()。他们可以运行 [`go get` 命令]() 来获取最新版本，也可以指定特定版本，如下例所示：```
$ go get example.com/mymodule@v0.1.0
```
